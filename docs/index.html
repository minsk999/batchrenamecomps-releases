폴더 정리까지, 한 패널에서.

BatchRenameComps는 After Effects · Premiere Pro의 컴포지션/시퀀스 이름을 규칙대로 일괄 변경하고, 폴더트리 생성·소스 자동 분류·프로젝트 저장·렌더까지 한 패널에서 처리하는 확장(익스텐션)이에요.

After Effects Premiere Pro Windows · macOS CEP 익스텐션
01

시작하기 - 설치

아래에서 ZIP을 받아 압축을 풀고, OS에 맞는 설치 파일을 더블클릭하면 돼요. 관리자 권한 없이 확장 폴더 복사와 CEP 디버그 모드 설정까지 한 번에 처리됩니다. (ZIP 안엔 스크립트 .jsx 버전도 함께 들어 있어요.)

BatchRenameComps v1.0.1
Windows · macOS 설치 패키지 (ZIP) · 스크립트 .jsx 포함
다운로드 ZIP
  1. 위 다운로드 버튼으로 ZIP을 받아 풀어요 (예: 다운로드 폴더).
  2. 폴더 안의 install_windows.bat을 더블클릭해요. (관리자 권한 필요 없어요)

  4. 메뉴 창(Window) → 확장(Extensions) → BatchRenameComps로 패널을 열어요.

02

처음 설정

설정(우측 상단 톱니 또는 마우스 4·5번 버튼)에서 두 가지만 잡아두면 바로 써요.

① 폴더트리 생성 위치

로컬 폴더트리 생성 위치에 작업 폴더가 만들어질 상위 경로를 지정해요. 비어 있으면 폴더 버튼을 처음 누를 때 탐색기로 고르게 돼요.

② 파일 임포트 폴더

소스를 가져올 기본 폴더 경로예요. 임포트 버튼이 이 폴더를 프로젝트 폴더트리로 분류해 넣어요. (필요할 때만 지정)

i해상도 프리셋·경로·폴더트리·단축키 등 설정은 설정 백업(내보내기/가져오기)으로 팀원과 공유할 수 있어요. 자세한 건 아래 설정 백업 항목을 보세요.
03

이름 구조

컴프 이름은 아래 순서의 토큰을 언더스코어(_)로 이어 만들어요. 빈 필드는 자동으로 건너뜁니다.

날짜_ 광고주_ 캠페인_ 페이지_ 작업내용_ 해상도_ 작업자_ @작업종류_ 작업구분
날짜 - 6자리(YYMMDD). 예: 260520
광고주 / 캠페인 / 작업내용 - 자유 입력
페이지 - 3p처럼 숫자+p
해상도 - 예: 1080x1920
작업자 - 이름(영문/한글)
@작업종류 - @ED(편집) 등 / 작업구분 - xian(시안)·vari(베리)·sf·retuch1 등 맨 끝 토큰
260520_KOZ_보이넥스트도어_3p_사운드편집_1080x1920_재현_@ED_xian
i@작업종류는 @MB(모션배너)·@BC(배너결합)·@SZ(사이즈배리)·@ED(편집)·@AV(AI활용영상)·@AB(AI활용배너)·@MV(모션영상)·@SH(촬영)·@CP(용량배리) 등을 드롭다운에서 골라요.
04

이름 변경 - 핵심 기능

각 필드를 채우고, 개별 적용(↻)하거나 한 번에 일괄 변경해요.

  • 각 필드의 ↻ 버튼 - 그 필드 하나만 선택된 컴프에 개별 적용해요.
  • 캠페인 / 페이지 / 작업내용의 ↻ - 첫 클릭은 현재 입력값으로 적용, 같은 값으로 다시 누르면 끝 숫자가 +1 돼요 (예: 캠페인1 → 캠페인2, 페이지는 1 → 2).
  • 불러오기 - 선택된 컴프 이름을 분석해 각 필드를 자동으로 채워요.
  • 이름변경 - 모든 필드를 조합해 선택된 컴프 이름을 한 번에 바꿔요.

우클릭으로 더 강력하게

  • 불러오기 우클릭 - 컴프를 선택하지 않아도, 프로젝트의 모든 컴프 중 이름순 가장 위 컴프를 자동으로 불러와요.
  • 이름변경 우클릭 - 컴프를 선택했으면 선택분만, 선택이 없으면 프로젝트의 모든 컴프 이름을 한 번에 변경해요. 이때 각 컴프의 기존 페이지·작업구분(시안/베리/리터치 등)은 그대로 유지돼요.
iPremiere Pro에서는 시퀀스 이름이 같은 방식으로 변경돼요.

06

폴더트리 생성 (폴더 버튼)

폴더 버튼 좌클릭 한 번이면 프로젝트 안 정리와 디스크 폴더 생성이 한꺼번에 돼요.

  1. 프로젝트 폴더트리 먼저 생성 - 현재 선택된 프로젝트(ae) 트리 구조대로 AE 패널 안에 폴더를 만들고, 기존 소스(푸티지·컴프·폴더)를 확장자/타입별로 자동 분류해요.
  2. 최상위 폴더 이름을 물어봐요 (기본값은 오늘 날짜).
  3. 로컬(디스크) 폴더트리 생성 - 지정한 위치에 실제 폴더를 만들고, 현재 프로젝트를 Ae/Pr 폴더에 자동 저장해요. 이 작업파일은 저장 기록에도 올라가요.
i우클릭 메뉴에서는 프로젝트 트리만, 또는 로컬 트리만 따로 실행할 수도 있어요. 좌클릭은 "한 번에 다", 우클릭은 "골라서"예요.

자동 분류 규칙

  • 이미지(jpg·png·psd·ai 등) → 트리의 Image 계열 폴더
  • 영상(mp4·mov·mxf 등) → Video 계열 폴더
  • 사운드(mp3·wav 등) → 03_Mix 계열
  • 컴프 → 01_Comp/_Main ( [comp] 태그가 가리키는 폴더 )
  • 매핑되지 않은 나머지 → Stock 폴더로 모음
iOverlord · TweiNa 폴더 예외 - 이름에 Overlord 또는 TweiNa가 들어간 폴더는 (피그마·포토샵·일러스트 연동 폴더라) 내부 구조를 그대로 유지한 채 통째로 Image 폴더로 들어가요. 같은 단어가 든 컴프는 01_Comp/Etc로 분류돼요. 빈 프로젝트 폴더트리 생성과 폴더 임포트 양쪽에서 동작해요.
07

임포트 · 분류

외부 폴더의 소스를 프로젝트 폴더트리에 맞춰 가져오면서 자동으로 정리해요. (After Effects 전용)

  • 임포트 버튼(좌클릭) - 설정의 임포트 폴더를 가져와 분류해요.
  • 임포트 우클릭 - 탐색기로 폴더를 직접 골라요 (설정 경로 무시, 프로젝트 저장은 안 함).
  • 폴더는 통째로, 낱개 파일은 개별로 - 최상위 폴더는 하위 구조까지 함께 대표 확장자 위치로, 흩어진 파일은 확장자별로 분류돼요.
  • 중복 방지 - 같은 파일은 다시 임포트하지 않고(패널 중복 방지), 디스크에 같은 이름이 있으면 덮어써요.
  • 제외 - 이름이 _Master·_Ae·_Pr로 끝나는 폴더는 분류에서 제외돼요.
08

렌더

컴프를 선택하고 렌더 버튼을 누르면 그 컴프만 바로 렌더돼요. 아무것도 선택하지 않으면 '프로젝트의 모든 컴프 N개를 렌더할까요?' 확인창이 떠서 [전체 렌더] 또는 [취소]를 고를 수 있어요.

그다음, 설정의 자동 출력 토글에 따라 동작이 둘로 나뉘어요.

자동 출력 ON - 렌더 버튼을 누르면 즉시 렌더하고, 결과물을 00_Master 폴더에 컴프 이름으로 저장해요. (렌더 중 AE가 잠깐 멈춰요)
자동 출력 OFF - 렌더 큐에 추가만 해요. 출력 위치 지정과 렌더 시작은 직접. 렌더 버튼 우클릭으로 00_Master 폴더를 열 수 있어요.
!After Effects는 출력 경로에 한글이 있으면 깨지는 고질적 제약이 있어요. 그래서 자동 출력은 내부적으로 영문(ASCII) 임시 폴더에 렌더한 뒤 결과물만 00_Master로 옮기는 방식으로 우회해요. 만약 임시 폴더 경로에까지 한글이 끼면(드문 경우) 안내창이 떠요 - 그때는 자동 출력을 끄고 렌더 큐에서 직접 출력 위치를 지정하면 돼요.
09

폴더트리 편집

설정에서 트리 구조와 확장자 매핑을 자유롭게 바꿀 수 있어요.

트리 종류

  • 프로젝트 폴더트리 - AE/Premiere 패널 안의 폴더 구조
  • 로컬 폴더트리 - 실제 디스크에 만드는 폴더

노드 편집

  • + 하위 폴더 추가 · ✎ 이름 변경 · 🗑 삭제(하위 포함)
  • 드롭다운 또는 닷(●) 인디케이터로 트리 전환

확장자 자동 분류 (ext 버튼)

프로젝트 폴더트리의 각 폴더에 확장자를 지정하면, 임포트·분류 때 그 확장자 파일이 해당 폴더로 들어가요. 특수 태그 [comp]는 컴포지션을 의미해요. (프로젝트 폴더트리 전용)

드래그 계층 이동

  • 노드 위 1/4: 위쪽에 삽입 · 아래 1/4: 아래쪽에 삽입
  • 노드 가운데 1/2: 자식으로 들어감 · 빈 영역: 최상위 맨 끝
i기본값 복원은 기본 4개 트리에만 쓸 수 있고, 확장자 매핑도 함께 복원돼요. 트리를 편집하면 폴더 버튼 우클릭 메뉴가 즉시 갱신돼요.

설정 백업

팀원 간 동일한 환경을 맞출 때 유용해요.

내보내기

해상도 프리셋·경로 설정·폴더트리를 JSON 파일로 저장해요.

가져오기

저장한 JSON에서 설정을 불러와 적용해요. 한 사람이 트리를 잡아두고 팀에 배포하기 좋아요.

익스텐션엔 있는데 스크립트엔 없는 것

  • 우클릭 보조 명령 전부 - 불러오기 최상위 자동, 이름변경 전체 컴프, 저장 최근 목록 열기, 렌더 00_Master 열기, 폴더 골라 실행. 스크립트에선 모두 불가예요(이름변경은 선택분만, 저장은 매번 이름 입력).
  • 폴더트리·확장자 매핑 편집 - 익스텐션은 노드 추가·삭제, 확장자 매핑, 드래그로 트리를 바꿔요. 스크립트는 트리·매핑이 코드에 고정(quick/full 두 종류만).
  • Overlord · TweiNa 폴더 예외 - 연동 폴더를 통째로 Image에 넣는 특수 처리는 익스텐션에만 있어요.
  • 저장 기록 - 최근 저장 파일 목록에서 골라 열기, 목록 비우기/닫기 UI는 익스텐션 전용이에요.
  • 모던 UI - 다크모드, 커스텀 폰트, 반응형 레이아웃은 ScriptUI에선 안 돼요. 입력창은 OS 기본 폰트로 표시돼요.
  • 인앱 가이드 - 익스텐션 안 도움말 대신, 이 설명서를 봐요.
!렌더 자동 출력은 00_Master가 이미 있어야 동작해요(없으면 안내창). 폴더/로컬 트리로 먼저 트리를 만들고 그 안에 프로젝트를 저장한 뒤 렌더하세요.
13

자주 묻는 질문

패널이 안 보여요

설치 후 AE를 완전히 종료했다 다시 켜요(작업 표시줄/Dock에서 완전 종료). 그래도 없으면 CEP 디버그 모드가 켜졌는지, 확장 폴더에 파일이 들어갔는지 확인해요.

패치(.ps1) 실행 시 실행 정책 오류가 나요

설치는 더블클릭(.bat/.command)이라 정책 설정이 필요 없지만, 패치 .ps1을 직접 실행할 땐 PowerShell에서 먼저 Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass를 실행한 뒤 패치를 실행하면 돼요. 사내용이라 정상이에요.

렌더 결과가 안 나오거나 "디렉토리가 없다"고 떠요

자동 출력은 현재 프로젝트가 폴더트리 안에 저장돼 있어야 00_Master를 찾을 수 있어요. 먼저 폴더 버튼이나 저장으로 프로젝트를 트리에 저장해두세요. 임시 폴더 경로에 한글이 끼었다는 안내가 뜨면 자동 출력을 끄고 큐에서 직접 출력 위치를 지정하면 돼요.

업데이트는 어떻게 하나요

전달받은 패치 .ps1을 실행하면 돼요. manifest가 안 바뀌는 패치는 패널만 닫았다 열면 적용돼요. 설정(트리·경로·해상도 등)은 그대로 유지돼요.

!문제가 계속되면 팀장(구민석)에게 화면과 증상을 알려주세요.
